# ✅ Answer Key

### Section A — True/False
- **A1. False** — reversed: ethics = external/social; morals = internal/personal.
- **A2. True.**
- **A3. True.**
- **A4. False** — that's consequentialism; deontology judges by **duty**.
- **A5. True** — a code of conduct is specific, enforceable rules + penalties.
- **A6. False** — that's *availability*; **integrity** = preventing unauthorized **modification**.
- **A7. True.**
- **A8. False** — a trademark protects **brand identifiers**; an algorithm is protected (if at all) by patent/trade secret.
- **A9. False** — legitimate interests **cannot** be used for **sensitive** data.
- **A10. True.**

### Section B — Multiple Choice
- **B1. A** — *ethos* = character (*mos* = custom → morals).
- **B2. C** — Altruism (favours all except the actor); egoism = self, utilitarianism = the group.
- **B3. B** — Emotivism.
- **B4. D** — the mean / what a virtuous person would do.
- **B5. A** — code of ethics (values); conduct = specific rules.
- **B6. C** — Availability.
- **B7. B** — Hash function (one-way digest).
- **B8. D** — sign with **private**, verify with **public**.
- **B9. A** — Historical (data generation); aggregation/learning = model building; deployment = deployment.
- **B10. C** — Intimacy (not monitored).
- **B11. B** — Institutional privacy (the organisation's data).
- **B12. D** — Utility (process/machine); design = ornamental, plant = new variety.
- **B13. A** — Trade secret (no disclosure, no expiry); patent needs disclosure + ~20 yrs.
- **B14. B** — Protection of public health/safety or the life/health of individuals.
- **B15. C** — Establish chains of responsibility & accountability (A, B, D are the inversions).

### Section C — Short Questions
- **C1.** **Confidentiality** = no unauthorized disclosure · **Integrity** = no unauthorized modification (status quo) · **Availability** = no unauthorized withholding (access when needed).
- **C2.** Code of ethics = high-level **values/principles** that govern **decision-making**; code of conduct = specific **rules** that govern **actions/behaviour** (and usually list discipline).
- **C3.** *Control of external influences:* **Solitude, Anonymity, Intimacy**. *Control of personal information:* **Reserve**.
- **C4.** *Data generation:* Historical, Representation, Measurement. *Model building:* Aggregation, Learning, Evaluation. *Deployment:* Deployment bias.
- **C5.** Trade secret, Trademark/Service mark, Patent, Copyright. **Patent ≈ 20 years.** Patent types: **Utility, Design, Plant.**
- **C6.** **Symmetric** = one shared secret key for encrypt + decrypt (fast). **Asymmetric** = public + private key pair. Symmetric's problem = **key distribution** (passing the key securely).
- **C7.** **Controller** decides the **purposes & means** (accountable); **processor** acts **on the controller's behalf**. The **controller** must notify SDAIA **within 72 hours** (and notify affected data subjects without undue delay).
- **C8.** **Egoism** = own interests above all · **Utilitarianism** = the group's greatest good · **Altruism** = good if consequences favour everyone except the actor.
- **C9.** Transparency, Accountability, Individual Agency, Data Privacy.
- **C10.** Any two of: (1) to **behave** like responsible professionals; (2) to **avoid** computer abuse/catastrophes; (3) because technology keeps creating **temporary policy gaps**.

### Section D — Case Studies *(model points — award marks for issue + theory application + judgment)*

**D1.**
- *Issue/stakeholders:* consent-free mining of students' public data for dropout prediction; students (privacy/autonomy), at-risk students, university, staff.
- *Deontology:* data wasn't given for this purpose → breaches consent/autonomy; **against** unless a genuine duty of care + transparency are honoured.
- *Utilitarianism:* justified **if** it reliably helps at-risk students at low privacy cost; false positives, stigma, and lost trust may outweigh.
- *Virtue ethics:* the intent to help is good, but prudence/respect favour **transparent, proportionate** methods (the mean), not covert surveillance.
- *Judgment:* **conditionally permissible** — only with transparency, data minimization, notice/consent where feasible, and PDPL compliance; covert mining fails.

**D2.**
- *SDAIA duty:* the **Controller** must notify **SDAIA within 72 hours** of a breach likely to harm data subjects, **notify affected customers** without undue delay, then **contain and document** (the three stages). A quiet fix is **non-compliant**.
- *Deontology:* honesty + customers' **right to know** → disclose.
- *Consequentialism:* concealment risks fraud, penalties, and lost trust that outweigh short-term reputation → disclose + mitigate.
- *Judgment:* **notify SDAIA and customers, contain, remediate.** Concealment is impermissible.

**D3.**
- *Harm sources:* **Historical bias** (20 yrs of biased approvals); possibly representation/measurement; **deployment bias** if used as-is.
- *Utilitarianism:* aggregate "gains" mask severe harm to the minority group → likely **net-negative** once equity/trust are counted.
- *Justice/deontology:* systematically disadvantaging a group violates the **duty of justice/fairness** regardless of totals → **impermissible**.
- *Judgment:* **don't deploy as-is** — rebalance data, validate on a representative population, run a fairness audit, keep human oversight.

**D4.**
- *(i) Secret recommendation algorithm:* a **trade secret** → reusing it breaches confidentiality/NDA → **violation**.
- *(ii) Curated customer database:* a curated/organised database **can be protected** → copying likely **infringes** (plus privacy/PDPL issues).
- *(iii) Reverse-engineering a public app:* generally **allowed** against a trade secret **unless** a patent/licence/contract bars it → **not a violation by itself**.
- *Deontology:* honesty + promise-keeping (NDA) → (i) and (ii) breach duty; (iii) (absent contract/patent) does not.
- *Virtue ethics:* integrity/fairness → exploiting a former employer's secrets is dishonest/unjust.
- *Judgment:* **stop using A's trade secret and database**; reverse-engineering is defensible only if unencumbered by patent/licence/contract.

**D5.**
- *Issue/stakeholders:* **security vs usability/other values** (patient care); patients, doctors, hospital, security team.
- *Tension:* cybersecurity is costly and consumes resources; **usability concerns can't justify weakening** security, **but** finding the right **balance** (resource allocation) is itself an ethical issue — weigh **harms, benefits, rights, and values**.
- *Best practices:* consider the **human lives** behind the system, **design for security AND usability**, **invite stakeholder (clinician) input**, and plan for **crisis/emergency** access.
- *Judgment:* don't weaken security outright; adopt a **usable-secure** solution (e.g., MFA + audited emergency "break-glass" access), involving clinicians and weighing harms/benefits before deciding.
